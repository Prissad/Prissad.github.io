    Blackjack - Reinforcement Learning

    Read More

    This is a Personal Project. It is an example of implementing a Reinforcement Learning Algorithm into a Game.

    The first step was to choose a Game to work on. For that purpose, I chose Blackjack while simplifying its rules.

    Using Python's Object Oriented Programming, I coded the game rules and logic by insisting on the concepts of state and action.

    Then came the step of creating the Reinforcement Learning Model using Q-Learning. Having created states and actions, I added a reward system depending or win/loss situation.

    Next I implemented 2 Sampling Methods: Monte-Carlo Control and Temporal Difference (or SARSA), and made multiple tests by changing the number of played games and other parameters to build the model and compare the Q value variance.

    I converted my work to a REST API and hosted it and I'm currently developing the visual Game that will consume the API.

  • Studies June 2021

    Peer-to-Peer Chat Platform

    Read More

    This Project was part of my Cybersecurity Class. It aims to create a safe End-to-End Chat environment.

    Peer-to-peer (or End-to-End) messaging is when data is encrypted by one user and sent to another user who's the only one able to decrypt it.

    My team and I created a Chat Web Application connected to an LDAP user registry itself connected to a Certificate Authority Server.

    A new user has a pair of keys created, the public one gets certified by the Certificate Authority and Stored in the Registry, while the private key is stored locally.

    When a user Bob wants to message a user Alice, a new Web Socket is created and when Bob sends a message that message gets encrypted by Alice's public key that is retrieved from the User Registry (LDAP) and sent inside the Socket. When the message is received by Alice, her private key is used to decrypt the data.

    Tools used:

    • Vue
    • NestJS
    • Trello
    • Git
